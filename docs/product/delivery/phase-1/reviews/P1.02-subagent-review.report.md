**Finding 1 — Unit route silently mismaps lessons at 4-part slug depth**

File: `src/pages/[subject]/[unit]/[lesson].astro`, line 13

**Advisory Observations**

**A1 — `unit: ""` passes schema validation and silently routes as standalone.** `z.string().optional()` accepts the empty string. An author who writes `unit: ""` instead of omitting the field entirely will have the lesson routed by the standalone page (because `!""` is `true`), not the unit page. If the MDX is physically inside a unit directory, `parts.slice(1).join('/')` produces a multi-segment `lesson` param (e.g., `lesson: 'unit-1/variables-and-io'`), generating an unusual URL. Adding `.min(1)` to the Zod `unit` field — `z.string().min(1).optional()` — would make empty-string units a build-time error and eliminate this ambiguity.

**A2 — Rationale states `lessons` lands in `DataEntryMap`; this appears imprecise for a `type: 'content'` collection.** In Astro 5, `type: 'content'` collections are typed under `ContentEntryMap`; `DataEntryMap` is reserved for `type: 'data'` (JSON/YAML) collections. The explicit `Array<CollectionEntry<'lessons'>>` annotation is correct and present in both route files, so the workaround functions regardless of the stated reason. The Rationale note may reflect an early Astro 5 type-inference quirk encountered during development; it does not affect runtime behavior, but a future reader auditing the type annotation might be confused by the stated justification. If the note is updated, "TypeScript inference for `getCollection` return type requires explicit annotation" is more accurate than the `DataEntryMap` claim.

**A3 — Standalone route has no depth guard for 1-part slugs.** `parts.slice(1).join('/')` on a 1-part slug yields an empty string `lesson` param. This only occurs if an MDX file is placed directly in `src/content/lessons/` rather than in a subject subdirectory (e.g., `src/content/lessons/orphan.mdx`). No current content is mis-placed, but the error mode is silent at the filter level — Astro would encounter the empty-string param at route-generation time. Parallel to the fix in Finding 1, a `parts.length === 2` guard in the standalone `filter` step would make mis-placed content a loud build-time failure.
