**Actionable findings**

1. **`src/content/lessons/programming-fundamentals/unit-1/variables.mdx` (Fragments at lines 34/64/68) and `src/content/lessons/programming-fundamentals/what-is-a-program.mdx` (Fragments at lines 33/51/55), root-caused in `src/pages/[subject]/[unit]/[lesson].astro` and `src/pages/[subject]/[lesson].astro`.**
   What's wrong: both routing pages call `entry.render()` and wrap the *entire* rendered MDX body in one `<Fragment slot="explanation"><Content /></Fragment>`, while independently populating `hook`/`analogy` slots from frontmatter fields. Astro/MDX only projects a content file's internal `slot="..."` Fragments into a layout when the file declares `layout:` in its own frontmatter (neither MDX file here does) — these two files were manually rendered instead. The result: the MDX body's `explanation` and `summary` Fragments are never delivered anywhere. This is empirically confirmed by reading the already-built `dist/programming-fundamentals/unit-1/variables/index.html` and `dist/programming-fundamentals/what-is-a-program/index.html`: the `<section>` for "Explanation" is completely empty (none of the Python code examples or "computer works the same way" prose appears), and the "Summary" section renders the layout's fallback placeholder text `[Summary placeholder]` instead of the authored summary copy. The `interactive` Fragment is likewise dropped (harmless only because it's an empty placeholder).
   Which clause: breaks invariant 3, and is a demonstrable correctness gap (clause 2) against the ticket Outcome ("All lesson layout slots are filled: hook, analogy, explanation, interactive (placeholder), summary").
   Fix recommendation: this is a pre-existing routing-layer gap (the `[lesson].astro` files aren't in this ticket's file list) exposed for the first time by real content. Two options for the primary agent: (a) fix `[subject]/[lesson].astro` / `[subject]/[unit]/[lesson].astro` to stop dumping the whole rendered `<Content />` into a single `explanation` Fragment and instead pass each MDX slot through correctly (or add `explanation`/`summary` as real frontmatter-driven fields the way `hook`/`analogy` already are), or (b) switch these MDX files to Astro's native frontmatter `layout:` pattern so the in-body `Fragment slot="..."` markup is actually respected. Either way, this needs to be resolved before the ticket can be considered to satisfy its stated Outcome — right now most of the authored lesson content is invisible on the live pages.
