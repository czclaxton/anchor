### Invariants to hold

1. Prev/next links must NOT be rendered as `<a>` elements on the first lesson (no prev) and last lesson (no next) respectively — an `<a>` to an undefined or incorrect lesson is a broken link.
2. Breadcrumb hierarchy must correctly reflect the current page: standalone lesson shows `Home > Subject > Title`; unit-grouped lesson shows `Home > Subject > Unit > Title`; subject landing shows `Home > Subject`.
3. Every lesson reachable from the home feed must also be reachable from its subject landing page — no lesson can appear on the home feed but be absent from its subject's listing.

### Attack surfaces to probe

1. `LessonLayout.astro` `findIndex` edge case — `allLessons.findIndex((l) => l.slug === entry.slug)` returns `-1` if no match. With `currentIndex = -1`: `prev` evaluates to `null` (correct) but `next` evaluates to `allLessons[0]` because `-1 < allLessons.length - 1` is true. If a slug mismatch occurred (e.g., slug encoding difference), the first lesson would erroneously appear as "next" on every page.
2. `Breadcrumb.astro` last-item rendering — the condition `i < items.length - 1 && item.href` renders non-last items with hrefs as links and everything else as `<span>`. Verify: if a middle item (e.g., unit) has no `href` or `href = undefined`, it silently renders as `<span>` instead of a link, potentially breaking breadcrumb navigation.
3. `LessonLayout.astro` unit breadcrumb href — unit breadcrumb items use `href: /${subject}` (links back to subject, not a dedicated unit page). Verify this matches stated behavior and doesn't create a link that points to a nonexistent URL.
4. `[subject]/index.astro` `getStaticPaths` completeness — subjects are derived from `new Set(entries.map(e => e.data.subject))`. If a lesson has a subject that contains no other lessons, the subject landing page is still generated. Verify all lessons on the home feed also appear on their respective subject page (Invariant 3).
5. URL construction `/${lesson.slug}` — Astro content collection slugs are auto-generated from file path relative to the collection. For `programming-fundamentals/what-is-a-program.mdx`, slug = `programming-fundamentals/what-is-a-program`, URL = `/programming-fundamentals/what-is-a-program`. For `programming-fundamentals/unit-1/variables-and-io.mdx`, slug = `programming-fundamentals/unit-1/variables-and-io`, URL = `/programming-fundamentals/unit-1/variables-and-io`. Verify the constructed hrefs match the route file URL patterns (`[subject]/[lesson]` vs `[subject]/[unit]/[lesson]`).
