### Attack surfaces to probe

1. **`runPythonExtractVars` — PyProxy object handling**: Pyodide returns Python-native objects (strings, ints) as JS primitives in most cases, but certain Python types may return as PyProxy objects requiring `.toJs()`. The code checks for `.toJs` method and calls it. Probe: does a Python integer (e.g. `npc_count = 4`) return as a JS number directly, or as a PyProxy? If PyProxy, does `val.toJs()` return the right primitive? Could `.toJs()` throw?

2. **`formatPythonError` — NameError message matching**: The regex `/NameError.*name '(\w+)' is not defined/` must match the actual Pyodide error message format. Pyodide 314.0.0 may format NameError differently across Python versions. Probe: what is the exact NameError message from Pyodide 314 for `season = summer`? Does the regex match it?

3. **`validateVars` — missing variable handling**: If the user deletes one of the 4 variable assignments from the editor, `pyodide.globals.get(name)` returns `undefined`. The `validateVars` check catches this and returns an error message. Probe: does the error message correctly identify which variable is missing, or does it fall through to a confusing path?

4. **Phaser scene initialization race**: `gameRef.current` is set inside the `.then()` callback of `import('phaser')`. If the user clicks "Update Scene" before Phaser finishes loading, `gameRef.current?.scene?.getScene('Park')` returns `undefined`, `scene?.updateVars` is not called, and no error is shown — the click is silently dropped. Probe: is this a correctness gap (invariant 1 says user should see an error), or is it acceptable UX (Phaser loads in ~2s, Pyodide takes longer)?

5. **`variables.mdx` import path resolution**: The MDX file is at `src/content/lessons/programming-fundamentals/unit-1/variables.mdx`. The import is `from '../../../../components/VariablesInteractive.astro'`. Count: 4 levels up from unit-1/ → programming-fundamentals/ → lessons/ → content/ → src/, then components/. That is actually 5 levels, but the import says 4. Probe: does the path correctly resolve to `src/components/` or does it resolve to `src/content/components/` (wrong)? Verify the build output — the build succeeded, suggesting it resolved correctly, but confirm the level count.
