**Actionable findings**

1. **`src/components/VariablesInteractive.tsx:524-525`** — Silent no-op when the Phaser scene isn't ready. `gameRef.current?.scene?.getScene('Park')` returns `undefined` if Phaser hasn't finished its dynamic import yet, and `scene?.updateVars(newVars)` then does nothing — no error is set, the button briefly shows "Running…" and reverts, and the user has no indication anything failed. This directly breaks Invariant 1 ("any failure ... must surface to the user as a visible error message rather than silently no-oping the scene update") for the "scene not ready" failure mode, which the invariant's own phrasing ("silently no-oping the scene update") describes almost verbatim. A related, narrower window also exists: even after `gameRef.current` is set, Phaser schedules `create()` asynchronously, so `getScene('Park')` can return a scene object whose `this.bg`/`this.treeLayer`/etc. aren't yet assigned; calling `updateVars` in that window throws inside the `try` block, which is not caught (only Python errors are caught explicitly), producing an unhandled promise rejection instead of a shown error. **Fix:** after resolving `scene`, check both `scene` truthiness and an explicit readiness flag (e.g., set `sceneReady = true` at the end of `create()`), and if not ready, call `setError('Scene is still loading — try again in a moment.')` instead of silently returning.

2. **`src/components/VariablesInteractive.tsx:100-101`** (`formatPythonError` fallback) — For any Python error that isn't a NameError and doesn't literally contain `"SyntaxError"` or `"invalid syntax"` — most notably `IndentationError` (a very plausible beginner mistake, e.g. an accidental leading space before a variable assignment) — the fallback picks the first non-blank line of the raw Pyodide message, which is the traceback header `Traceback (most recent call last):`, not the actual exception. The user sees `Python error: Traceback (most recent call last):`, which is meaningless and undermines the ticket's "syntax errors → readable syntax hint" goal (invariant 2c) for a realistic input. **Fix:** find the *last* non-blank line of the traceback (or search for a line matching `/^\w*Error: /`) rather than the first, and/or widen the syntax-error substring check to include `IndentationError`/`TabError`.
